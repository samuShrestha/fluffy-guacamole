A Quick Introduction

Websites, like this one, consist of many interlocking pieces that work together to form everything that you see.

3 core pieces that make up a website are HTML, CSS, and Javascript.

HTML is a form of markup language that allows for the developer to define the layout and content of a website.

Cascading Style Sheets (CSS) are what allow the developer to “stylize” their content. CSS can help define the font, size, color, spacing, border and location of HTML information on a web page.

Javascript is a scripting language that runs within your browser that allows for more extensive user interaction.

Data!

Furthermore, when creating web apps such as Social Networks like Facebook, or services like Netflix, such applications have to deal with data. Whether it may be your friend’s Relationship Status, family photos, or how many hours you’ve wasted watching “educational” documentaries, all of it’s stored in something known as a database. Often, these applications have to interact with such data, adding, changing, or even deleting particular information. Within the world of web development there exists a multitude of “conventional” options to go about integrating and manipulating data within applications, but recently, there’s been a new player on the field.

Javascript, the traditional client-side (running on your computer) scripting language, has made a huge jump to the server side (running on the server), thanks to a technology known as Node.JS, and has begun to catch fire amongst developers.

Using Node.js is...

More Universal

The awesome part about running Node.js is that both the server and client-side code is written in one language and can also be run on multiple platforms (desktops, mobile devices, etc.).

Cross-Platform Compatible

Javascript can run on both desktop and mobile platforms, allowing an application to programed only once and then deployed to multiple device platforms.

More Efficient

Not only is utilizing Javascript as a server-side language simply faster, but merging the front-end and back-end into a single codebase makes development much faster!

Better Suited for Real-Time Data

Node.js’s event-driven architecture makes real-time applications a lot easier, again due to a singular codebase, making the synchronization process much more potent and economical.

Backed by a Large Community

The Node.JS community is ever expanding and has a large collection of already user-made modules (reusable code) that can be used by anyone!

However, it’s important to note with as amazing as Node.js seems, it isn’t an all-in-one silver bullet. Node.js is a solution to a particular need and understanding this is absolutely essential. You definitely don’t want to use Node.js for intensive computation (scientific data perhaps), that of which nearly eliminates all of Node.js’s advantages. Yet where Node.js really shines is in creating fast, versatile applications that utilize client-to-server interactions.

   <input id="m" autocomplete="off" /><button>Send</button>

<script>
   var socket = io();
   $('form').submit(function(){
      socket.emit('chat message', $('#m').val());
      $('#m').val(''); return false;
   });
   socket.on('chat message', function(msg){
      $('#messages').append($('<li>').text(msg));
   });
</script>
Server-Side Code 
var app = require('express')();
var http = require('http').Server(app);
var io = require('socket.io')(http);

app.get('/', function(req, res){
  res.sendFile(__dirname + '/index.html');
});

io.on('connection', function(socket){
  socket.on('chat message', function(msg){
    io.emit('chat message', msg);
  });
});

http.listen(3000, function(){
  console.log('listening on *:3000');
});

So, what does all of this necessarily mean?

Node.js is opening whole new doors for the web development industry. Currently, Javascript is the only language that is able to interact with the client (your computer), and with its actively growing community and wealth of technologies, it’s evident that Javascript is here to stay. But for now, Javascript is simply a trend with large potential. When server-side Javascript was introduced to the industry, it brought a whole lot of excitement and opportunity with it, but calling Javascript the future of web development may be a tad far-fetched. But the Javascript development stack (a development “process” in which a developer utilizes Javascript for the core aspect of an application, rather than a singular facet) is fairly young, and the future is open-minded, so Javascript has a clear start but still a long way to go.
